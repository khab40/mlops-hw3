## Phase 1: vLLM Serving Configuration

Model: `Qwen/Qwen3-30B-A3B-Instruct-2507`

Endpoint: `http://localhost:8000`

Launch command: `scripts/start_vllm.sh`

| Flag | Value | Justification |
|---|---:|---|
| `--model` | `Qwen/Qwen3-30B-A3B-Instruct-2507` | Required assignment model for final serving and evaluation. |
| `--host` | `0.0.0.0` | Binds the OpenAI-compatible server on the VM so SSH port forwarding can expose it to the laptop. |
| `--port` | `8000` | Matches the assignment's expected vLLM endpoint and Prometheus scrape target. |
| `--max-model-len` | `8192` | Avoids the default 262k context, which exceeded available KV cache memory on one H100; still covers the expected 1.5-3K token prompts plus short SQL outputs. |
| `--gpu-memory-utilization` | `0.90` | Gives vLLM most of the H100 memory while leaving some headroom for CUDA/runtime overhead. |

## Phase 2: Observability Core

Dashboard: `infra/grafana/provisioning/dashboards/serving.json`

Grafana URL: `http://localhost:3000/d/vllm-serving/vllm-serving`

Prometheus scrape target: `host.docker.internal:8000/metrics`

The Phase 2 dashboard uses the metrics exposed by the running vLLM OpenAI-compatible server:

| Area | Panels | Why it matters |
|---|---|---|
| Request volume | Requests served, HTTP request rate | Confirms traffic is reaching vLLM and separates normal completions from aborts/length stops and HTTP errors. |
| Scheduler pressure | Scheduler queue | Shows running vs waiting requests so load-induced queueing is visible. |
| Throughput | Token throughput, Request token shape | Separates prompt prefill tokens from generated decode tokens and shows whether prompts/outputs changed during a test. |
| Latency | Chat completion latency, Time to first token, Request lifecycle latency | Tracks API p50/p95/p99 plus lifecycle p95 for queue, prefill, decode, and total running time. |
| KV cache | KV cache headroom, Prefix cache hit ratio, KV eviction pressure | Shows cache utilization, reusable prefix-cache benefit, and preemptions that indicate cache/scheduling pressure. |
